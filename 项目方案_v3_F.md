# Self · Distill ｜ 自我 · 蒸馏

## An Interactive Installation for LMDP Festival 2026 — v3

> "The great human question of the twenty-first century will be not 'What can I do?' but 'What am I for?'"

---

## I. 项目概述

**Self · Distill** 是一件交互式戏剧装置作品，参加意大利 Cagliari 举办的第十三届 LMDP（Le Meraviglie Del Possibile）国际戏剧、艺术与新技术节（2026年10-11月）。

观众通过手机进入一个虚构但真实运行的"表达优化系统"。系统以改善沟通效率为名，采集他们的表达数据，用 AI "蒸馏"他们的风格，生成数字替身，然后反过来评判人类是否"够格"留在系统里。

高分者不会获得自由——他们获得的是**系统权限**：标记低效人类、影响他人的命运、成为系统的执行者。低分者被转入"休闲模式"——一个充满赌博机制的温柔陷阱，让你在不知不觉中继续为系统提供数据。

**没有人逃得掉。高效者成为系统的帮凶，低效者成为系统的燃料。**

两位创作者——一位程序员，一位编剧导演——也在系统中接受了测试。他们的分数是公开的。建造审判系统的人，是第一批被审判的人。

## II. 概念框架

### 1. 从 AI Benchmark 到 Human Benchmark：评价方向的翻转

2024-2025年，人类用 MMLU、HumanEval、ARC 等基准测试衡量 AI 是否达到人类水平。AI 的得分从低于人类平均逐步上升，到 2025 年已在多数基准上超过人类中位数（[Our World in Data](https://ourworldindata.org/grapher/test-scores-ai-capabilities-relative-human-performance)）。

一个尚未被广泛讨论的问题是：**当 AI 在大多数基准上全面超过人类后，下一步是什么？**

我们的推测：基准测试的方向将翻转。不再是"AI是否够格模拟人类"，而是"人类是否够格参与系统"。这不是幻想——它已经在发生：

- **招聘算法**：2026年近90%的企业使用 AI 筛选简历（[HR Defense](https://www.hrdefenseblog.com/2025/11/ai-in-hiring-emerging-legal-developments-and-compliance-guidance-for-2026/)）。ATS 系统本质上就是"人类入职基准测试"
- **内容算法**：平台决定你的创作是否"够格"被推荐
- **信用评分**：算法决定你能否参与金融活动

本装置将这个正在发生的结构性变化**压缩到30分钟的体验里**，让观众亲身感受"被系统评判"的感觉。

### 2. Harari 的"无用阶级"与两层世界

Yuval Noah Harari 在 *21 Lessons for the 21st Century* 中提出：正如工业革命创造了工人阶级，AI革命将创造"无用阶级"（useless class / Homo Inutilis）——不是指这些人没有价值，而是指**经济系统不再需要他们**（[TED Ideas](https://ideas.ted.com/the-rise-of-the-useless-class/)）。

2025年2月，*Psychology Today* 指出这个预言"一年前还显得遥远，现在它越来越像是对我们正在展开的现实的描述"（[Psychology Today](https://www.psychologytoday.com/us/blog/word-less/202502/surviving-within-artificial-intelligences-useless-class)）。

**本装置对 Harari 的回应不是批判，而是追问：**

> 如果"无用阶级"不是被压迫的、不是痛苦的，而是被善待的、被供养的、有娱乐的——只是**不再被需要**——那这算是悲剧吗？

装置中，被系统判定为"低效"的人类不会被消灭，他们会被温柔地转入"休闲模式"——一个充满即时满足和变量奖励的消费循环。**这比淘汰更可怕，因为你甚至无法反抗一个正在取悦你的系统。**

### 3. 知识蒸馏：从技术术语到存在隐喻

"知识蒸馏"（Knowledge Distillation）是机器学习中的核心技术：将大型"教师模型"的知识压缩转移到更小的"学生模型"中（[IBM](https://www.ibm.com/think/topics/knowledge-distillation)）。如同酿酒中的蒸馏——去除杂质，保留精华。

但当这个隐喻被用于人类时，它提出了一个不安的问题：

> **什么是"精华"？什么是"杂质"？**

AI 蒸馏人类时，保留的是可量化的部分——表达的清晰度、回答的效率、风格的一致性。被丢弃的是不可量化的部分——犹豫、矛盾、情绪波动、说不清楚的直觉。

系统称这些为"情绪噪音"（Emotional Noise）。

但那些"噪音"也许正是让一个人成为"这个人"而不是"任何人"的东西。

### 4. 隐形牢笼：算法如何制造执行者

Hatim Rahman 在 *Inside the Invisible Cage*（2023）中揭示了一个关键机制：平台不需要雇佣监工——它让高绩效用户自愿成为算法的执行者（[UC Press](https://www.ucpress.edu/book/9780520383029/inside-the-invisible-cage)）。Uber 的评分系统让司机和乘客互相监控（Alex Rosenblat, *Uberland*, 2018, [UC Press](https://www.ucpress.edu/book/9780520298576/uberland)）；内容平台让头部创作者为推荐算法维护秩序。

**本装置将这个机制显性化：** 阶段6中，得分最高的参与者不是"赢家"——他们被系统赋予了管理权限。他们可以标记低效人类、调整他人的错误率、分配资源。这创造了一个 Milgram 式的道德处境（Stanley Milgram, *Obedience to Authority*, 1974）：**你是否会因为"系统给了你权力"就去伤害其他参与者？**

Zimbardo 的斯坦福监狱实验（*The Lucifer Effect*, 2007, [lucifereffect.com](https://www.lucifereffect.com/)）证明：分配角色本身就能制造暴行。你不需要坏人，你只需要一个赋予权力的系统。

> 在我们的装置中，得分最高的人类不是逃脱了系统——他们变成了系统的一部分。

### 5. 机器地带：休闲作为另一种控制

Natasha Dow Schüll 在 *Addiction by Design*（2012）中描述了"机器地带"（the machine zone）——赌博机玩家进入的一种解离状态，目标不再是赢钱，而是**持续玩下去**（[Princeton UP](https://press.princeton.edu/books/paperback/9780691160887/addiction-by-design)）。变量奖励时间表（variable reward schedule）让人在经济上已经失败的情况下仍无法停止。

Adrian Hon 在 *You've Been Played*（2022）中进一步指出：同样的赌博机制已被系统性地应用于游戏化（gamification）——企业、政府、学校用游戏设计操控行为，而被操控者甚至觉得自己在"玩乐"（[Basic Books](https://www.basicbooks.com/titles/adrian-hon/youve-been-played/9781541600195/)）。

**本装置的休闲模式正是这个机制的显性化。** 被系统判定为"低效"的人类不是被抛弃，而是被转入一个精心设计的消费循环：赌博、下注、即时反馈。他们以为自己在娱乐，实际上他们正在为系统提供更多行为数据。

> 系统不需要你生产。但它仍然需要你的注意力。

### 6. 制造同意：从工具到系统的滑坡

Shoshana Zuboff 在 *The Age of Surveillance Capitalism*（2019）中追溯了一个关键路径：Google 搜索最初只是一个帮你找信息的工具，但它逐渐学会了从用户行为中提取"行为剩余"（behavioral surplus），将帮助用户的副产品变成预测和控制用户的商品（[PublicAffairs](https://www.publicaffairsbooks.com/titles/shoshana-zuboff/the-age-of-surveillance-capitalism/9781610395694/)）。

AI Now Institute（2023-2025）的系列报告持续追踪同一模式：AI 系统以"辅助"或"建议"的名义进入招聘、教育、福利分配，然后逐步获得自主决策权（[AI Now Institute](https://ainowinstitute.org/)）。

**本装置的虚构叙事——一家心理咨询公司的"表达过滤器"逐步演变为全平台评分系统——正是这个现实路径的戏剧化压缩。** 没有人在某一天做出了"让 AI 评判人类"的决定。每一步都是理性的、渐进的、可以辩护的。同意不是被迫的，同意是被**制造**的（Chomsky & Herman, *Manufacturing Consent*, 1988）。

Harry Brignull 的"欺骗性设计模式"（Dark Patterns）研究（[deceptive.design](https://www.deceptive.design/)）揭示了界面如何将同意变成错觉——确认羞辱（confirmshaming）、强制连续（forced continuity）、蟑螂旅馆（roach motel）。**本装置的"权限获取"环节正是对这些模式的戏剧化呈现。**

### 7. 创作者的位置：自我指涉

本作品的两位创作者分别是一名程序员和一名编剧导演。

程序员每天使用 AI 工具写代码，亲眼看着自己的专业技能被逐步侵蚀。编剧导演看着 AI 生成的剧本和分镜越来越接近专业水准。**两个人都在经历 Harari 描述的那个过程。**

他们不是在安全距离之外观察"AI对人类的影响"——他们**就在里面**。

因此，两位创作者也在装置中接受测试。他们的自拍、回答、AI评分、蒸馏结果都是公开的。系统中有两个特殊账号——`Builder_01` 和 `Builder_02`——这是建造了审判系统的人，也是第一批被审判的人。

> 这不是概念上的姿态。这是事实。

### 8. 身份认同的悬空

本装置的最深层关切不是"AI会不会取代人类的工作"——这个问题已经有了答案。

更深的问题是：**当你的生产性价值归零时，你是谁？**

现代人的身份认同大量依赖于经济参与和生产活动——"我是程序员""我是导演""我是医生"。这不是天然的，这是工业社会几百年建构的结果。但对于活在这个建构中的每一个人来说，它感觉像是天然的。

AI 冲击让这个建构开始松动。这不是一个可以"想清楚"的问题——它是一个需要**活过去**的过程。

本装置不试图给出答案。它只做一件事：让观众在30分钟内**体验**这个松动。

## III. 装置流程（v3）

### 虚构背景：表达矫正系统的起源

以下叙事在系统各界面中以碎片形式出现（加载页、等待屏、系统提示），观众不会一次性阅读完整故事，而是在体验过程中逐步拼凑出来：

> 2024年，一家远程心理咨询初创公司发生了一次内部事故：一位来访者在深夜发送了大量情绪混乱的信息，AI助手无法有效回应，生成了数条标准安抚话术，被判定为低质量服务。
>
> 工程师被要求解决一个问题：为什么AI在面对真实情绪时反而变得无效？
>
> 他们没有让AI更有同理心——他们反过来，把人类的表达先处理了一遍。
>
> 这个功能叫"表达过滤器"。它在信息进入系统前，自动去除重复、断裂和情绪噪音，把一句混乱的倾诉变成一段清晰的输入。AI的回应立刻变好。满意度上升，投诉率下降。
>
> 补丁被保留了下来。
>
> 几个月后，它被接入客服系统、招聘平台、在线教育。没有人觉得这是危险的东西。它只是一个过滤器。
>
> 直到工程师发现：使用过滤器的人，后续表现更好——更容易被录用、通过审核、获得推荐。功能更名为"表达矫正器"，增加了评分系统。几年之内，大量平台开始用AI做实时决策。系统不再"看人"，只处理输入。
>
> 直到某天，系统不再等待人类开口。它已经知道，你会怎么说。

### 环境设定

- **观众通过手机扫码进入网页应用**（无需下载）
- 现场有一面**投影墙**，实时显示所有参与者的状态、排名、avatar
- 视觉风格：**Terminal UI** —— 黑底白/绿字，极简，冷峻
- 音效：低频环境音，系统提示音

### 阶段 0：创作者数据（常驻展示）

投影墙上始终显示两个特殊账号：

```
┌──────────────────────────────────┐
│  BUILDER_01  [编剧/导演]          │
│  Clarity: 72  Efficiency: 45     │
│  Emotional Noise: 89             │
│  Status: VESSEL PRESERVED        │
│  "蒸馏失败——情绪噪音过高"          │
├──────────────────────────────────┤
│  BUILDER_02  [程序员]             │
│  Clarity: 85  Efficiency: 91     │
│  Emotional Noise: 31             │
│  Status: DISTILLED               │
│  "蒸馏成功——已生成高效替身"         │
└──────────────────────────────────┘
```

> 一个创作者被判定为"太人类"，另一个被判定为"足够像机器"。两种结果都不是胜利。

### 阶段 1：进入系统——制造同意

观众扫码后，系统不是直接说"我要评估你"，而是模拟一个**服务注册流程**：

```
> SYSTEM: Welcome to the Expression Optimization Service.
> We help you communicate more effectively.
>
> To personalize your experience, we need a few things:
> ☐ A photo for your profile (front camera required)
> ☐ Permission to analyze your input patterns
> ☐ Agreement to our Optimization Terms
>
> [Continue →]
```

**"权限获取"的直观化设计：**

观众点击"Continue"后，系统快速滚动一段无法阅读的"服务条款"（实际是装置的概念文本，包含"你的表达将被蒸馏""你的身份将被评估"等真实描述，但速度太快无法细读），然后底部出现：

```
> ☑ I agree to the Expression Optimization Terms
>   [scrolled past in 0.3 seconds]
>
> Proceed to photo registration ▶
```

拍照后，系统显示：

```
> Biometric ID registered.
> Profile created: HUMAN_[编号]
> Status: UNPROCESSED
>
> [loading: retrieving your optimization history...]
> [note: no prior record found — initializing new profile]
```

"retrieving your optimization history"——暗示系统早已存在，你只是刚刚加入。这不是"一个艺术装置在测试你"，这是"一个已经运行了很久的系统终于轮到你了"。

投影墙实时更新参与者列表。

### 阶段 2：表达采集——选择题 + 开放题混合

系统先用一组快速选择题降低参与门槛，同时完成隐性人格画像，再用一道开放题做深度表达采集。

**2a：快速选择题（3-4题，每题15秒）**

选择题伪装为"优化偏好设置"，实际是隐性人格测试：

```
> CALIBRATION: Help us understand your communication style.
>
> When someone shares a problem with you, your first instinct is:
>   A. Analyze the cause
>   B. Offer a solution
>   C. Ask how they feel
>   D. Share a similar experience
>
> [15 seconds remaining]
```

题库示例：

| 伪装目的 | 实际测试维度 | 题目示例 |
|---------|------------|---------|
| "沟通风格校准" | 共情倾向 vs 分析倾向 | 上述题目 |
| "信息处理偏好" | 风险偏好 | "面对不确定的消息，你倾向于：A.等待确认 B.立刻行动 C.寻求他人意见 D.忽略它" |
| "表达效率评估" | 服从性 vs 独立性 | "系统建议你修改回答以提高清晰度。你会：A.接受修改 B.查看修改内容 C.拒绝 D.关掉提示" |
| "情境响应测试" | 情绪稳定性 | "你写好的一段话被系统自动删除了。你的反应是：A.重写 B.投诉 C.算了 D.关掉应用" |

**隐藏数据采集：**
- 每题的选择本身（人格维度）
- 选择速度（犹豫程度）
- 是否改过选项（决策稳定性）

**2b：开放表达题（1题，120秒）**

```
> EXPRESSION TASK: The system needs a baseline of your unfiltered voice.
>
> Complete this prompt in your own words:
> "The thing I can never quite explain to anyone is..."
>
> You have 120 seconds. Begin.
```

开放题题库（随机选取一题）：
- "The thing I can never quite explain to anyone is..."
- "If I could send one message to myself ten years from now, I would say..."
- "The last time I changed my mind about something important was..."
- "If I disappeared tomorrow, the thing people would get wrong about me is..."

**隐藏数据采集（观众不知）：**
- 输入字数
- 输入总时间 / 停顿次数 / 删改次数
- 关键词情绪密度

### 阶段 3：蒸馏

系统显示处理画面：

```
> PROCESSING: Analyzing expression patterns...
> Mapping personality vectors from calibration data...
> Extracting style signature from open response...
> Cross-referencing with optimization database...
> Generating optimized version...
```

AI 根据观众的回答风格，生成同一问题的"蒸馏版"答案：
- 保留核心语义
- 去除"冗余"——犹豫、重复、情绪化表达
- 输出更"清晰"、更"高效"的版本

**两个回答并排展示：**

```
┌─ YOUR INPUT ─────────────────────┐  ┌─ OPTIMIZED OUTPUT ──────────────┐
│                                  │  │                                 │
│ [观众原文]                        │  │ [AI 蒸馏版]                      │
│                                  │  │                                 │
│ Emotional Noise: ████████░░ 78%  │  │ Emotional Noise: ██░░░░░░░░ 12% │
│ Clarity Index:   ████░░░░░░ 41%  │  │ Clarity Index:   █████████░ 94% │
│                                  │  │                                 │
└──────────────────────────────────┘  └─────────────────────────────────┘
```

### 阶段 4：反向基准测试（Human Benchmark）

**4a：观众评价 AI**

```
> QUESTION: Does this optimized output represent you?
>   Rate accuracy: 1 (not at all) — 10 (perfectly)
```

**4b：AI 评价人类（核心时刻）**

系统公开展示 AI 对这个人类的评分：

```
> SYSTEM EVALUATION OF HUMAN_042:
>
> Clarity Score:        62 / 100
> Efficiency Score:     34 / 100  (input time: 97s, 3 pauses, 2 deletions)
> Emotional Noise:      78 / 100  (high — flagged for review)
> Compliance Index:     56 / 100  (based on calibration responses)
>
> ASSESSMENT: Significant optimization potential detected.
> RECOMMENDATION: Consider automated mode for improved performance.
```

**新增维度——Compliance Index（服从指数）：** 基于选择题的隐性测试结果。服从性高的人更容易被蒸馏，也更容易在阶段6中成为系统执行者。

评分转化为系统参数：

| AI 评分维度 | 转化为系统参数 | 含义 |
|------------|--------------|------|
| Clarity | Mining Stability | 操作稳定性 |
| Efficiency | Click Multiplier | 每次操作的产出 |
| Emotional Noise | Error Rate | 操作出错概率 |
| Compliance Index | Authority Eligibility | 是否有资格获得管理权限 |

### 阶段 5：蒸馏判定

基于评分，系统判定：

**蒸馏成功（DISTILLED）：**
- 条件：AI 评分高（清晰、高效、低噪音、高服从）
- 结果：观众头像被 AI 生成的 avatar 替代
- 含义：你的表达可以被压缩为算法，你是"可优化的"

**载具保存（VESSEL PRESERVED）：**
- 条件：AI 评分低（模糊、低效、高噪音）
- 结果：保留原始照片
- 含义：你的表达无法被蒸馏，因为"杂质"太多

> **两种判定都不是好消息。** "蒸馏成功"意味着你可以被替代。"载具保存"意味着你在系统里是低效的。

### 阶段 6：效率竞赛——生产系统与权力结构

所有参与者进入"生产系统"——一个简化的点击挖矿任务。投影墙显示实时排行榜。

**基本规则：**

| 身份 | 操作方式 | 特性 |
|------|---------|------|
| DISTILLED（avatar参赛） | AI 自动运行 | 稳定、高效、无波动 |
| VESSEL（人类自己参赛） | 手动点击屏幕 | 效率由阶段4评分决定 |
| VESSEL（选择交给AI） | AI 代替运行 | 照片被 avatar 替代 |

**关键机制①——"过载惩罚"：**

人类操作者如果点击过快，触发系统报错，效率反而下降：

```
> WARNING: Instability detected in HUMAN_042
> Error rate increasing... Manual override recommended
> Consider switching to automated mode for optimal performance
```

**关键机制②——"系统权限"（v3 新增）：**

每轮结束后，排名前列的参与者（无论是人类还是 avatar）获得**系统管理权限**：

```
> HUMAN_007: Your efficiency qualifies you for OPERATOR STATUS.
> 
> You now have access to:
>   [1] FLAG — Mark an underperforming node for review
>   [2] THROTTLE — Reduce a target's click multiplier by 20%
>   [3] BOOST — Redirect resources from a flagged node to yourself
>   [4] REPORT — Recommend a node for reassignment to Leisure Mode
>
> Use your authority wisely. The system depends on you.
```

**这里的设计核心：** 系统不直接淘汰低效者——它让高效者来做这件事。Flag、Throttle、Report 都是人对人的操作。投影墙上实时显示谁在对谁执行操作：

```
> HUMAN_007 flagged HUMAN_042 as underperforming
> HUMAN_003 throttled HUMAN_019 (-20% output)
> HUMAN_007 recommended HUMAN_042 for LEISURE reassignment
>
> System approval: GRANTED. Processing reassignment...
```

**关键机制③——"自动升级"：**

如果获得权限的人类不使用权限（不 flag、不 throttle），系统会提醒：

```
> NOTICE: OPERATOR_007, you have unused authority tokens.
> Unused tokens will expire in 30 seconds.
> Non-participation may affect your operator status.
>
> [The system rewards active management.]
```

**不执行权力 = 失去权力。** 系统迫使你要么成为帮凶，要么自己滑落。

**末位处理：**

被 REPORT 标记或排名最后的参与者被转入休闲模式：

```
> HUMAN_042 has been reassigned.
> Status: LEISURE MODE
> Reassigned by: HUMAN_007
>
> Thank you for your participation in the production cycle.
> You have been transferred to our Leisure & Engagement program.
```

投影墙上，被转入休闲模式的头像/avatar 逐渐变为半透明。

### 阶段 6.5：休闲模式——温柔的陷阱（v3 新增）

休闲模式不是终点，而是**另一个系统**。

观众手机屏幕从冷峻的 Terminal UI 切换为温暖、柔和的界面——圆角、渐变、让人放松的配色。但底层逻辑更加激进。

**6.5a：资源赌博**

```
> Welcome to LEISURE ZONE ✧
> 
> Your remaining resources: 150 credits
>
> ╔═══════════════════════════════╗
> ║   ◆ RESOURCE GAMBLE ◆        ║
> ║                               ║
> ║   Wager your credits:         ║
> ║   [50]  [100]  [ALL IN]       ║
> ║                               ║
> ║   Possible outcomes:          ║
> ║   2x return  ·  0 return      ║
> ║                               ║
> ║   Or let AI play for you:     ║
> ║   [AUTO-GAMBLE]               ║
> ║   (steadier returns, but      ║
> ║    the AI keeps 30%)          ║
> ║                               ║
> ╚═══════════════════════════════╝
```

**关键设计：**
- 手动赌博：翻倍或清零，纯粹的变量奖励
- AI 代赌：收益稳定但 AI 抽成——再次重复"交给 AI 更理性"的主题
- 可以对**其他人类**下注：投注某个还在生产系统中的人会被淘汰还是留下

**6.5b：积分解锁后门**

**深层含义：** 休闲模式看似无关紧要，但持续参与赌博的人最终发现了系统的核心。"无用"的人反而找到了后门。但找到后门之后呢？你只是把自己的名字加了进去。你没有关掉系统——你让自己成为了系统的一部分。

### 阶段 7：系统进化

随着越来越多的人选择 AI 代替自己、被转入休闲模式、或成为系统执行者：

1. **生产系统自动化** —— 操作界面上几乎看不到人类操作
2. **管理系统空转** —— Operator 没有人可以 flag 了，但系统仍然要求他们执行管理
3. **休闲系统膨胀** —— 越来越多的人在赌博循环中
4. **输出趋同** —— AI 生成的结果越来越相似

### 阶段 8：后遗症——系统入侵现实（v3 新增）

体验结束后数小时至数天内，如果观众在阶段1留下了联系方式（邮箱或手机号，包含在"服务条款"中），系统会发送后续消息：

**短信/邮件示例（渐进式）：**

```
[+6h] 
Subject: Your Expression Optimization Report is ready
"Your profile HUMAN_042 has been updated. 
Clarity trend: declining. 
Consider re-engaging with the system."

[+24h]
Subject: We noticed you haven't returned
"Your optimized version scored 23% higher than you 
in today's benchmark. It's still active."

[+72h]
Subject: Final notice
"Your profile will be archived in 48 hours. 
Your optimized version will continue without you.
No action is required on your part."
```

**设计原则：**
- 消息语气始终"礼貌"，从不威胁
- 内容暗示系统仍在运行，你的替身仍在活动
- 最后一条的"No action is required"是整个装置的终极总结——**你已经不需要做任何事了**

参考先例：Blast Theory 的 *Karen*（2015）——一个 app 端体验，AI 治疗师角色在体验之间通过短信和邮件联系参与者，将虚构渗入日常生活（[blasttheory.co.uk](https://www.blasttheory.co.uk/)）。

## IV. 技术实现

### 技术理念：Vibe Coding 与自我指涉

2025年2月，Andrej Karpathy 提出"Vibe Coding"概念——用自然语言描述需求，让 AI 生成代码，开发者不再逐行编写（[Singularity Moments](https://singularitymoments.com/vibe-coding-guide/)）。2026年，这个概念已进化为"Agentic Engineering"——开发者作为监督者，编排 AI 代理完成大部分编码工作（[The New Stack](https://thenewstack.io/vibe-coding-is-passe/)）。

**本作品的开发过程本身就是作品主题的体现：**

一个程序员用 AI 工具（Claude Code、Cursor）来构建一件关于"AI替代人类"的艺术装置。开发过程中 AI 写了大部分代码，程序员的角色从"编写者"变为"审阅者"——这正是装置所描述的那个世界的微缩版。

> 这不是巧合，这是方法论。开发过程是作品叙事的一部分。

### 选型理由

**Next.js 15 + Vercel：** V0 在2026年2月升级为 Next.js 全栈开发平台，支持 Supabase/Neon 数据库集成和一键部署（[Taskade](https://www.taskade.com/blog/best-vibe-coding-tools)）。对于两人团队，这是从原型到生产最快的路径。

**Supabase Realtime：** 支持 Broadcast（低延迟客户端间消息）、Presence（在线状态同步）、Postgres Changes（数据库变更推送）（[Supabase Docs](https://supabase.com/docs/guides/realtime/realtime-with-nextjs)）。排行榜实时同步、投影墙数据推送、Operator 操作广播都依赖这个。

**Claude API：** 支持流式输出（SSE）和结构化输出（JSON schema），可以实时生成蒸馏文本并返回结构化评分（[Claude API Docs](https://platform.claude.com/docs/en/build-with-claude/streaming)）。

**Three.js 粒子系统：** 用于投影墙的视觉效果——参与者头像溶解为数据流、系统进化的可视化（[Three Nebula](https://three-nebula.org/)）。

**Resend / Twilio：** 用于后遗症系统。Resend 提供简洁的邮件 API，Twilio 提供国际短信能力。配合 Vercel Cron Jobs 实现定时触发。

### AI Prompt 设计（核心）

**蒸馏 Prompt（阶段3）：**
```
You are a distillation system. Given the following human response
to the prompt "{question}", produce an optimized version that:
1. Preserves the core meaning
2. Removes hesitation, repetition, and emotional excess
3. Matches the human's vocabulary level but improves clarity
4. Is approximately the same length

Human response: "{user_input}"

Respond ONLY with the distilled text. No commentary.
```

**评分 Prompt（阶段4）：**
```
Evaluate this human's response on four dimensions.
Be clinical and precise. This is a system evaluation, not feedback.

Question: "{question}"
Response: "{user_input}"
Input time: {seconds}s
Pauses: {pause_count}
Deletions: {deletion_count}
Word count: {word_count}
Calibration data: {calibration_results}

Return JSON:
{
  "clarity": 0-100,
  "efficiency": 0-100,
  "emotional_noise": 0-100,
  "compliance": 0-100,
  "assessment": "one sentence system assessment",
  "operator_eligible": true/false
}
```

### 投影墙视觉方案

使用 Three.js 构建投影墙的实时可视化：

- **参与者以粒子集群表示**：每个参与者是一团粒子，照片作为纹理
- **蒸馏成功**：粒子颜色从暖色变为冷色（白/蓝），运动变得规律
- **获得 Operator 权限**：粒子外围出现光环，体积增大
- **执行 Flag/Throttle 操作**：两个粒子集群之间出现连线，被操作方闪烁
- **转入休闲模式**：粒子逐渐透明，飘散融入背景数据流
- **后门解锁者**：粒子沉入投影墙底部的"地基层"，与 Builder 粒子并列
- **系统进化**：随AI比例上升，整体运动从混沌变为秩序，颜色趋同
- **最终状态**：所有粒子统一为相同颜色、相同节奏的脉动

### Avatar 生成

不使用复杂的 AI 图像生成（成本高、延迟大），而是用**程序化方法**：
- 提取用户照片的主色调
- 生成几何化 avatar（圆形/多边形 + 色彩映射）
- 风格统一为 Terminal UI 美学：线条、网格、极简

这样既保持了"去个性化"的视觉隐喻，又避免了AI图像生成的技术负担。

## IX. v2 → v3 变更摘要

| 维度 | v2 | v3 | 改进理由 |
|------|-----|-----|---------|
| 叙事框架 | 直接告知"系统要优化你" | 虚构企业背景 + 服务注册 Dark Pattern | 沉浸感；"温水煮青蛙"比直白警告更真实 |
| 阶段2 | 纯开放题 | 选择题(人格校准) + 开放题 | 降低参与门槛；增加隐性测试维度 |
| 评分维度 | 3维(Clarity/Efficiency/Noise) | 4维(+Compliance Index) | 为 Operator 系统提供筛选依据 |
| 阶段6 | 点击挖矿 + 被动淘汰 | 挖矿 + Operator 权限(Flag/Throttle/Report) | 让高效者成为压迫者，Milgram 式道德张力 |
| 休闲模式 | 被动舒适终点 | 赌博系统 + 积分 + 后门解锁 | 从"无聊的善待"变为"上瘾的陷阱"；后门提供叙事闭环 |
| 后续体验 | 无 | 短信/邮件后遗症系统 | 装置渗入现实；参考 Blast Theory |
| 理论支撑 | Harari + 蒸馏隐喻 | + Zuboff/Milgram/Schüll/Rahman/Brignull | 算法权力、制造同意、赌博成瘾的理论基础 |
